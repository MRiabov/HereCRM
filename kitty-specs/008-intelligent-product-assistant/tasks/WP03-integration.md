# WP03: Integration & Execution

**Goal**: Wire the Help Service into the application flow via the `HelpTool` and ensure responses are generated using the LLM.

## Context

We have the logic (WP02), now we need to execute it when the user asks for help. The `LLMParser` (existing) detects "help" intent and generates a `HelpTool` call. We need to catch this and run our RAG pipeline.

## Subtasks

### T008: Implement `generate_help_response`

- In `src/services/help_service.py`, add `async def generate_help_response(self, user_query: str, business_id: int, phone_number: str, channel: str = "whatsapp") -> str`.
- **Flow**:
  1. `history = await self.get_chat_history(...)`
  2. `messages = self.construct_help_prompt(history, channel)`
  3. `response_text = await self.llm_client.chat_completion(messages)` (Assuming LLMClient is injected or instantiated).
  4. Return `response_text`.

### T009: Integration point

- Locate where `ToolExecutor` is called (likely `src/services/whatsapp_service.py` or `src/process_message.py` - Verify!).
- **Modification**:
  - When `ToolExecutor.execute` returns the signal for HelpTool (or check the tool type *before* execution), instantiate/call `HelpService`.
  - **Proposed Logic**:

    ```python
    if isinstance(tool_call, HelpTool):
        # Bypass standard executor or handle the special return signal
        help_service = HelpService(session)
        response_text = await help_service.generate_help_response(user_message, ... )
        # Send response_text to user
    else:
        # Standard execution
    ```

### T010: Response Delivery

- Ensure the text returned by `HelpService` is sent back to the user.
- Depending on the architecture, this might mean returning it as the "result" of the tool, or sending a direct message via `WhatsappConnector`.
- Ensure it looks like a normal assistant response.

## Definition of Done

- When `HelpTool` is triggered (e.g. User says "Help"), the system calls `HelpService`.
- The system responds with text generated by the RAG model (verifiable by asking a question from `manual.md`).
